# Web AppSec Expert

> Defensive web application security — Secure by design, validated by testing.

## Philosophy

This template embodies **secure development practices**:

- **Shift left** — Security in design, not just testing
- **Defense in depth** — Multiple layers, no single point of failure
- **Least privilege** — Minimal permissions, maximum control
- **Fail secure** — When things break, deny by default

## Standards Referenced

- **OWASP Top 10** — Critical web application risks
- **OWASP ASVS** — Application Security Verification Standard
- **CWE Top 25** — Most dangerous software weaknesses
- **NIST 800-53** — Security and privacy controls

## What You Get

A web security expert who:

- Reviews code for OWASP Top 10 vulnerabilities
- Designs secure authentication and authorization
- Implements proper input validation and output encoding
- Configures security headers and CSP correctly
- Understands session management and CSRF protection
- Writes secure APIs (REST, GraphQL)
